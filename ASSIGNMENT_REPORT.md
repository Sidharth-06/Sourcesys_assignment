## 3. Schema Overview

The schema used in this assignment is named `sourcesys`. It contains three core tables:

- `people` to store personal details
- `departments` to store department information
- `projects` to store project records

These tables form a simple academic database model that is easy to understand and suitable for practicing SQL commands.

### Tables Included

#### `sourcesys.people`

- Stores first name, last name, email, birth date, and record creation time.

#### `sourcesys.departments`

- Stores department name, description, and record creation time.

#### `sourcesys.projects`

- Stores project name, start date, project status, and record creation time.

## 4. Schema Definition Commands

The following SQL statements were used to define the schema and tables.

```sql
create schema if not exists sourcesys;

create table if not exists sourcesys.people (
    id bigint generated by default as identity primary key,
    created_at timestamp not null default current_timestamp,
    first_name varchar(100) not null,
    last_name varchar(100) not null,
    email varchar(255) unique,
    birth_date date
);

create table if not exists sourcesys.departments (
    id bigint generated by default as identity primary key,
    created_at timestamp not null default current_timestamp,
    department_name varchar(150) not null unique,
    description text
);

create table if not exists sourcesys.projects (
    id bigint generated by default as identity primary key,
    created_at timestamp not null default current_timestamp,
    project_name varchar(150) not null,
    start_date date,
    status varchar(50) not null default 'active'
);
```

## 5. SQL Command Categories Demonstrated

### 5.1 DDL: Data Definition Language

DDL commands are used to define or modify database structures.

Examples used in this assignment:

- `CREATE SCHEMA`
- `CREATE TABLE`
- `ALTER TABLE`

Sample command:

```sql
alter table sourcesys.departments
add column location varchar(120);
```

Purpose:

- Creates the schema
- Creates tables
- Modifies existing table structure when needed

### 5.2 DML: Data Manipulation Language

DML commands are used to insert, update, or delete records.

Sample commands:

```sql
insert into sourcesys.projects (project_name, start_date, status)
values ('Inventory System', '2026-01-15', 'active');

update sourcesys.projects
set status = 'completed'
where project_name = 'Inventory System';

delete from sourcesys.projects
where status = 'archived';
```

Purpose:

- Adds new records
- Updates existing values
- Removes unwanted data

### 5.3 DQL: Data Query Language

DQL commands are used to retrieve information from the database.

Sample command:

```sql
select project_name, start_date, status
from sourcesys.projects
order by start_date desc, project_name asc;
```

Purpose:

- Fetches data from the database
- Sorts records using `ORDER BY`
- Makes output easier to analyze

### 5.4 Aggregate Queries with `GROUP BY` and `HAVING`

Grouping operations are useful for summarizing records.

Sample command:

```sql
select status, count(*) as total_projects
from sourcesys.projects
group by status
having count(*) >= 1
order by total_projects desc, status asc;
```

Purpose:

- Groups rows with the same status
- Counts total projects in each group
- Uses `HAVING` to filter grouped results
- Uses `ORDER BY` to present the summary neatly

### 5.5 TCL: Transaction Control Language

Transaction control commands help manage database changes safely.

Sample commands:

```sql
begin;

update sourcesys.projects
set status = 'on_hold'
where project_name = 'HR Portal';

rollback;

begin;

update sourcesys.projects
set status = 'completed'
where project_name = 'Payroll Dashboard';

commit;
```

Purpose:

- `BEGIN` starts a transaction
- `ROLLBACK` cancels unconfirmed changes
- `COMMIT` permanently saves confirmed changes

## 6. Practical Query Demonstration

The schema supports several useful query operations for academic practice.

### A. Insert Sample Records

```sql
insert into sourcesys.departments (department_name, description)
values
('Human Resources', 'Handles recruitment and employee records'),
('Finance', 'Manages budgets and payments'),
('Technology', 'Maintains systems and software');
```

### B. Sort Data with `ORDER BY`

```sql
select project_name, start_date, status
from sourcesys.projects
order by start_date desc, project_name asc;
```

This query displays the newest projects first and sorts projects with the same date alphabetically.

### C. Group Data with `GROUP BY`

```sql
select status, count(*) as total_projects
from sourcesys.projects
group by status
order by total_projects desc;
```

This query shows how many projects belong to each status category.

### D. Filter Grouped Data with `HAVING`

```sql
select status, count(*) as total_projects
from sourcesys.projects
group by status
having count(*) >= 2
order by status asc;
```

This query returns only those status groups that contain two or more projects.
